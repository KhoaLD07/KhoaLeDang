Stack and Calling Convention

In ASM when we SUB rsp we expand the size of the stack. But in this blog, for explaining I will use +8 when we push—in fact it will be -8 to rsp.

Example: bigSum(int a, int b, int c, int d, int e) returning a + b + c + d + e, called as bigSum(10, 9, 5, 1, 10):

// prep
mov [rsp + 0x20], 10
mov r9, 1
mov r8, 5
mov rdx, 9
mov rcx, 10   // rsp 0
call bigSum   // rsp = 8 -> 0

bigSum:       // 8
push rbp      // 16
mov rbp, rsp
sub rsp, 8    // 24
xor rax, rax
add rax, rcx
add rax, rdx
add rax, r8
add rax, r9
add rax, [rsp + 0x28]
add rsp, 8    // 16
pop rbp       // 8
ret           // 0

It's 0x28 (40 decimal) because 8 bytes per register 8 × 4 = 32 bytes (0x20), but when we call the function it pushes the return address, so the offset from the stack pointer is 0x28.

Another example: a function that calls another function. We use the Borrow and repay rule: when you call a function you have to align so that the return address offset is -8 from the call.

// print:       // 8
//   push rbp   // 16
//   mov rbp, rsp

Register

First of all, we need understand about Registers. Registers are located in the CPU, this is the fastest data storage inside a computer used by CPU to performs calculation almost immediately.

The Table below is the list of common registers in x64 architecture (Windows) you may encounter in assembly.

Name (64 bit)Full NameMain PurposeCalling Convention
RAXAccumulatorMath operation (+, -, *, /)Hold function's returned value.
RCX CounterCount variable for loops, and bit shifting.1st Arguments in function.
RDXDataLargeMath (*,/) 128 bit2nd Arguments in function.
R8Register 8It can do the math similar to RAX,RCX. Can be used while RAX,... are busy. 3rd Arguments in function.
R9Register 9Similar to R84th Arguments in function. The Stack Boundary, after R9 others storage will be push into STACK (RAM)(slower)
RSISource IndexPointer to String/ArrayCallee-saved(*)
RDIDestination IndexPointer to where RSI should be placed inCallee-saved
RBXBaseIt is used as a base memory address pointer.Callee-saved
RSPStack PointerPoints to the top of the stack. Last In - First Out (LIFO).Callee-saved
RBPBase PointerPoints to the bottom of the stack (Usually push rbp, mov RBP, RSP when enter a function, and then sub RSP, [StackSize] to allocate space for the function)Callee-saved
RIPInstruction PointerStore the Address of the next instruction to executed

(*):Callee-saved: (The function must preserve the value of this register; it should remain unchanged after the function returns)
int main() (1):The different between RBX and RSI is when you want to access to an element inside. RSI (use lodsq) will change but RBX will remain the same there.
